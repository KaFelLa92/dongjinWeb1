[ Spring 정적파일 ]
    - Spring 프로젝트내 HTML/CSS/JS/IMG 등 파일
    -> src -> main -> resources -> static 폴더 위치
        -> 만약에 다른 폴더내 html/css/js/img 등이 있을 경우 식별 불가
    1.  Spring 프로젝트는 정해진 폴더(static)내 HTML 경로 지원한다.
        http://localhost:8080/(static)이하 경로 html
        예] http://localhost:8080/example/example1.html
    2.  지원확장자 : .html .js .css .jpg 등 주로 정적파일이다

[ Spring 동적파일 ]
    -> Spring 프로젝트내 JSP(JAVA+HTML:MVC1) /타임리프(JAVA+HTML) /머스테치/REACT(JS+HTML:MVC2) 등등 파일
        JSP란? .jsp 파일내 java와 html 코드를 모두 작성 가능한 확장자 : MVC1패턴(controller+view)
            -> 강의기준은 .jsp 파일 사용하되 특정기능만 사용 , MVC2패턴(controller와 view 분리)
    -> 정적파일은 코드 그 자체로 브라우저 해석(렌더링) VS
    -> 동적파일은 코드 그 자체로 브라우저 해석(렌더링)불가능 , 각 프로그래밍 언어들 렌더링

[ Spring과 JSP 통합 ]
    1. 문제점 : Spring 회사는 공식적인 JSP 지원하지 않는다.
    2. 설치방법 : 그레이들 환경
        1) build.gradle 파일내 jsp 의존성(dependencies) 추가 -> ctrl+shift+O (새로고침)
            https://mvnrepository.com/artifact/org.apache.tomcat.embed/tomcat-embed-jasper 접속
            implementation 'org.apache.tomcat.embed:tomcat-embed-jasper:11.0.7' 복사
        2) 패키지 구성
            (1) ( src -> main -> webapp 경로 생성 )
            (2) webapp 폴더내 .jsp 파일 생성
            * 코드 작성은 VSCODE에서 합니다. 실행은 스프링, 인텔리제이에서.
            (3) 모든 .jsp 파일 상단에 jsp 설정코드
            <%@ page language = "java" contentType = "text/html; charset = utf-8" pageEncoding = "utf-8" %>
            * localhost:8080/(webapp) 이하 경로로 URL 정의된다.
            -> http://localhost:8080/test.jsp
